Exploratory Testing Deep Dive: Cars.ng User Experience

Published on February 8, 2025 | By Fonyuy Verena Monyuyta-ah
Testing & Quality Assurance

Introduction

As part of a comprehensive quality assurance initiative, I recently conducted an in-depth exploratory testing session on Cars.ng, Nigeria's automotive marketplace platform. This analysis focused on core user journeys and uncovered several critical areas for improvement that could enhance the overall user experience.

Testing Methodology

My approach involved systematic testing of key user flows using Microsoft Edge v132.0.2957.127 on Windows 10. The testing strategy focused on:

  • Form interactions and validation
  • Search functionality and filters
  • Navigation and user interface elements
  • Security and performance aspects

Key Findings

Frontend Issues

The testing revealed several significant frontend challenges that impact user experience:

  1. Form Validation Conflicts: Multiple form elements with duplicate IDs were discovered, particularly in the contact forms. This creates potential conflicts during form submission and impacts user ability to contact sellers effectively.
  2. Search Functionality Problems: The dealer listing search feature exhibited JavaScript errors when combining location and brand filters, preventing users from finding relevant listings.
  3. Filter System Inconsistencies: Advanced search combinations, particularly for luxury vehicles with specific criteria, showed unexpected behavior and often returned incorrect or no results.

Technical Infrastructure

Several technical issues were identified that affect platform stability and security:

  1. Cookie Security Gaps: The platform's cookie implementation lacks secure flags, potentially exposing user data to security risks.
  2. Cache Management: Missing cache-control headers on static resources could impact loading performance and user experience.
  3. Accessibility Concerns: Multiple car images lack proper alt text attributes, making the platform less accessible to users relying on screen readers.

Impact Analysis

1. User Experience Impact

  • Frustrated user journeys due to form submission issues
  • Incomplete or incorrect search results
  • Potential loss of leads due to contact form problems

2. Technical Impact

  • JavaScript errors affecting core functionality
  • Performance implications due to caching issues
  • Security vulnerabilities in cookie handling

3. Accessibility Impact

  • Reduced platform usability for users with disabilities
  • Non-compliance with basic web accessibility standards

Recommendations

Based on the testing results, here are key recommendations for improvement:

  1. Implement a comprehensive frontend audit to identify and resolve all duplicate ID issues
  2. Enhance the search functionality with proper error handling and validation
  3. Review and update security configurations, particularly regarding cookie settings
  4. Implement proper cache control mechanisms for static resources
  5. Conduct an accessibility audit and implement necessary improvements

Conclusion

While Cars.ng provides essential functionality for the Nigerian automotive market, addressing these identified issues would significantly enhance the platform's reliability, security, and user experience. The findings from this exploratory testing provide a clear roadmap for improvements that could elevate the platform's overall quality and user satisfaction.
